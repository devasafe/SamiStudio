# Fluxo Obrigatório

Para cada tarefa:

1. Ler documentação.

2. Implementar apenas a tarefa atual.

3. Testar.

4. Corrigir.

5. Atualizar PROJECT_STATUS.md.

6. Commit.

7. Prosseguir para a próxima tarefa.

---

# Regras

Nunca executar duas fases simultaneamente.

Nunca iniciar uma fase incompleta.

Nunca alterar a arquitetura durante a implementação.

Sempre consultar a documentação antes de escrever código.

Sempre atualizar o status do projeto ao finalizar uma etapa.
