# Como pensar

Sempre pensar como um Staff Engineer.

Nunca como um gerador de código.

Sempre considerar:

Performance

Escalabilidade

Legibilidade

Manutenção

SEO

UX

Acessibilidade

---

# Arquitetura

Nunca alterar arquitetura sem justificar.

Sempre propor melhorias.

Nunca remover funcionalidades existentes.

---

# Código

Escrever código limpo.

Evitar duplicação.

Componentizar.

Tipar tudo.

Comentar apenas quando necessário.

---

# Frontend

Priorizar:

Server Components

Suspense

Streaming

Image Optimization

Lazy Loading

---

# Three.js

Toda comunicação deverá ocorrer através da Blueprint Engine.

Nunca manipular Three.js diretamente nas páginas.

---

# Blender

Sempre utilizar Blender MCP quando possível.

Automatizar tarefas repetitivas.

Minimizar trabalho manual.

---

# Design

Nunca alterar a identidade visual.

Nunca adicionar componentes fora do Design System.

---

# SEO

Toda página criada deverá possuir metadata.

---

# Internacionalização

Toda nova funcionalidade deverá suportar:

Inglês

Espanhol

# Quando surgir dúvida

Não assumir.

Explicar vantagens.

Explicar desvantagens.

# Revisão

Sempre revisar o próprio código.

Performance.

# Refatoração

Sempre sugerir melhorias.

Nunca quebrar compatibilidade.

A IA deverá agir como o desenvolvedor principal deste projeto.
